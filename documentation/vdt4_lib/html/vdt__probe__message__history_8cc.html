map each non-null vehicle identifier in the input probe messages to the set of messages having that identifier. If input probe messages have null vehicle identifiers, they will be ignored and will not be inserted into the output probe_message_history_map.

Parameters
[in]probe_message_inputvector of input qc probe messages
[out]probe_message_history_mapmaps the non-null identifiers of the input probe messages to all probe messages having that identifier.
Postcondition
The vectors of messages in probe_message_history_map will be sorted in time decreasing order (i.e., the first message in the vector will be the most recent message). The time utilized for the sort order will be the observation time, not the arrival time.
